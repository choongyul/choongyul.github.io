October 13, 2020 in TIL, Programming

프로그래밍 언어론을 배워야 하는 이유

새로운 언어를 쉽게 배울 수 있다.

새로운 프로그래밍 언어를 학습하는 능력을 향상시키는 간단한 방법은 여러개의 언어를 배우는 것이다. 여러 언어를 배우면 새로운 언어에 익숙해 지는 시간은 서서히 줄어 들것이다. 그리고 언어의 차이란 단지 Syntax의 차이라고 생각할 것이다. 절차적 프로그래밍에 익숙한 C개발자가 FORTRAN, BASIC을 쉽게 학습하는 것에 빚대어 말이다.

정말 언어를 배우는 건 문법의 차이에 익숙해 지는 일일까? 우리는 같은 범주에 있는 언어를 배우며 스스로를 너무 높게 평가하는 것이 아닌지 생각해 보아야 한다. 절차지향 프로그래밍에서 객체지향 프로그래밍으로 패러다임을 바꾸는 일은 쉽지않다. CppCon 2015에서 Kate Gregory가 말했듯 C언에 익숙한 사람은 표현만 C++할 뿐이지 완벽하게 객체지향 적으로 프로그램을 구성할 수 없다. C++가 C언어 문법을 수용하기 때문이라고 말할 수 있지만 C언어를 배운사람이 바로 Java를 배운다고 바로 객체지향적 사고와 설계 능력을 가질 수 없다. Java 엔지니어니어 객체지향적으로 사고할 수 있게 된 배경에는 Spring Framework이 주요한 역할을 했다. 객체지향적으로 사고 할 수 밖에 없도록 틀이 제공되었기 때문이다.

프로그래밍 언어론에서는 언어의 발전과 패러다임 변화를 살펴보며 새로운 언어에 접근하는 법을 알려준다.
