프로그래밍 언어론 - Day5

October 13, 2020 in TIL, Programming

프로그래밍 언어론을 배워야 하는 이유

새로운 언어를 쉽게 배울 수 있다.

새로운 프로그래밍 언어를 쉽게 학습하는 간단한 방법은 여러개의 언어를 배우는 것이다. 여러 언어를 배우면서 새로운 언어에 익숙해지는 시간을 서서히 줄이는 방법이다. 다양한 언어를 접하는 경험이 늘어나면, 언어의 차이란 단지 Syntax의 차이라고 생각할지도 모른다.

언어를 배우는 건 문법의 차이에 익숙해지는 것일까? 우리는 같은 범주에 있는 언어를 배우며 스스로를 너무 높게 평가하고 있지는 않은지 생각해 보아야 한다. 난 학부생 때 C를 배우 난 후 C++를 배웠다. 처음에는 서로 문법이 호환되니 C++도 별다른 것 없다고 생각했다. 하지만 Object와 Class가 개념이 나오면서 내 생각이 틀렸다는 걸 알게 되었다. 절차지향 언어에서 객체지향 언어로 패러다임을 바꾸는 일은 쉽지 않았다. 회상해보면 객체지향 언어 JAVA, C++를 배운 건 대학생 때지만, 객체지향적으로 프로그래밍을 한다고 느꼈을 때는 스프링 프레임워크를 사용하고 난 후였다. 프레임워크 틀안에서 기능을 구현하려고 하니 객체지향적으로 사고할 수밖에 없었다.

프로그래밍 언어를 배운다는 건 다른 형태의 표현을 배우는 것이 아니다. 한국말로 나는 개발자입니다.를 영어로 I am a developer.라고 번역하는 것처럼 JAVA로 구성된 프로그램을 C#으로 변경하는 것으로 생각하면 안 된다. 새로운 언어를 배우는 건 언어의 담긴 사상을 이해하고 그 사상에 맞춰 생각하는 법을 배우는 일이다. JAVA Lambda Expression을 배우는 것이 단순히 Anonymous Inner Class를 Lambda Expression으로 바꾸는 법을 배우는 건 아니지 않는가?

프로그래밍 언어를 쉽게 배우려면 공통적으로 사용되는 구문과 의미에 대해 알아야 한다. 개발 언어 패러다임의 변화와 이유를 연결 지을 수 있어야 한다. 컴퓨터 발전의 역사를 공부하며 100년 전 수학자들이 만든 사상이 어떻게 프로그래밍 적으로 현실화되고 있는지에 관심을 가져야 한다. 그리고 자신의 사용하는 주력 언어의 발전방향을 예상할 수 있어야 한다. 이런 안목을 얻기 위해 고리타분한 프로그래밍 언어론을 공부하자! 먼지 덮인 책을 펼칠 때 날아가는 먼지가 빛을 받아 반짝이는 순간이 나의 영감이 빛나는 순간으로 만들어 보자!
